# R21 Typed Control Gate Training Contract

R21 does not train final answer text. It trains and evaluates typed control decisions that happen before answer generation.

The required labels are:

- `response_type`
- `response_mode`
- `binding_kind`
- `question_type`
- `operation`
- `active_referent`
- `topic_shift_kind`
- `repair_eligibility`
- `answer_density`
- `verifier_expected`

Failure records must enter the failure bank before runtime patches. A failure may only become a runtime change after it has been generalized into a scenario family with train, dev, and blind coverage.

R21 split rules:

- Splits are by scenario family and entity family, not random rows.
- Train/dev/blind must not share the same entity family.
- Blind rows must include entities or contexts not used in train.
- A passing report must include confusion matrices, binding accuracy by kind, repair precision/recall, affordance overtrigger, generic fallback illegal count, same-template streak, duplicate answer rate, unique scenario families, blind-only families, and runtime entity-specific branch counts.

Anti-overfit rules:

- No new `web/dialog_rules.js` case patches.
- No answerIndex expansion to fix current failures.
- No new `web/tiny_router_model.generated.js` patch unless generated by a validated training export.
- No new entity-specific branches in the controller, contextual resolver, response mode manager, topic stack, or conversation state inference.
- Existing entity-specific debt in older culture answer surfaces must be counted and reported until migrated.
