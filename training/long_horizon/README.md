# Long-Horizon Task Scaffold

R24A freezes further model training until the project can measure whether the runtime keeps task state, constraints, context, and useful judgment across turns.

This directory is for behavioral training and evaluation samples, not factual knowledge expansion. Samples should describe observable user goals, constraints, turns, expected behavior, forbidden behavior, provenance, and scoring rules.

Do not store chain-of-thought. Short rationales, behavior labels, and scoring rubrics are allowed when they are observable and reviewable.

Required provenance:
- `human_seed`: project-authored seed rows.
- `synthetic_llm`: future imported rows generated by an external model. Must name `generator_model` and must be reviewed before training use.
- `repo_derived`: derived from repository docs or code. Must include license or permission notes.
- `eval_fixture`: small test fixtures. Must include license or permission notes.

Rows with private data are not allowed unless explicitly reviewed, and reviewed private rows must not enter public runtime or public training artifacts.
